코드는 이미 저장소에 있어요(`supabase/functions/naver-callback/`). 아래만 하면 됩니다.

```
https://ifjikwpmyfvzwetppnfx.supabase.co/functions/v1/naver-callback
```

## ① 네이버 개발자센터 — 앱 등록

1. [developers.naver.com](https://developers.naver.com) 로그인 → **Application → 애플리케이션 등록**
2. 애플리케이션 이름: `물들임`
3. **사용 API**: **네이버 로그인** 선택
   - 제공 정보(권한): **회원이름 · 이메일 · 별명** 체크
4. **환경 추가 → PC 웹**
   - 서비스 URL: `https://icebrain78.github.io`
   - **네이버 로그인 Callback URL**:
     ```
     https://ifjikwpmyfvzwetppnfx.supabase.co/functions/v1/naver-callback
     ```
5. 등록 완료 → **Client ID / Client Secret** 복사

---

## ② Edge Function 배포 (대시보드 방식, CLI 불필요)

1. Supabase → 왼쪽 **Edge Functions** → **Deploy a new function**
2. 함수 이름: `naver-callback`
3. 저장소의 `supabase/functions/naver-callback/index.ts` 내용을 **그대로 붙여넣기**
4. **"Verify JWT" 옵션을 끕니다(OFF)** ← 중요 (네이버가 브라우저로 부르는 거라 인증헤더 없음)
5. **Deploy**

> CLI가 있다면: `supabase functions deploy naver-callback --no-verify-jwt`

---

## ③ 시크릿(비밀값) 등록

Supabase → **Edge Functions → (naver-callback) → Secrets** (또는 Project Settings → Edge Functions):

| 이름 | 값 |
|------|-----|
| `NAVER_CLIENT_ID` | 네이버 Client ID |
| `NAVER_CLIENT_SECRET` | 네이버 Client Secret |

> `SUPABASE_URL`, `SUPABASE_SERVICE_ROLE_KEY` 는 자동으로 들어있으니 안 넣어도 됩니다.

---

## ④ 앱에 Client ID 넣기 (공개 OK)

`js/cloud-config.js` 의 `naverClientId` 에 네이버 **Client ID** 를 넣고 커밋:

```js
window.CLOUD_CONFIG = {
  url: "https://ifjikwpmyfvzwetppnfx.supabase.co",
  anonKey: "…",
  naverClientId: "여기에_네이버_Client_ID",
};
```

> 이 값이 채워지면 로그인 창에 **"네이버로 계속하기"** 버튼이 나타납니다.
> (Client Secret은 절대 여기 넣지 마세요 — Edge Function 시크릿에만.)

- `error=session_failed` → 서비스롤 문제(보통 자동주입이라 드묾)
- 로그인 후 갤러리로 안 돌아옴 → Authentication → URL Configuration의 Redirect URLs에
