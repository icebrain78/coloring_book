2. **API 및 서비스 → OAuth 동의 화면**

3. **API 및 서비스 → 사용자 인증 정보 → OAuth 클라이언트 ID**
   - 유형: **웹 애플리케이션**
   - **승인된 리디렉션 URI**: `https://ifjikwpmyfvzwetppnfx.supabase.co/auth/v1/callback`
   - → **클라이언트 ID / 보안 비밀** 복사

## 2. 카카오 로그인 ⚠️ (콘솔이 개편돼서 위치가 특이함)

### 2-A. 앱 생성 + 비즈 앱 전환
1. [developers.kakao.com](https://developers.kakao.com) → 애플리케이션 추가 (이름 `물들임`)
2. **앱 아이콘 등록** (`앱 기본 정보 → 앱 아이콘`) — 비즈 전환에 필요
3. **비즈 앱 전환** — 개인 개발자도 **본인인증**만으로 무료 (사업자번호 불필요)
   - **이메일 동의는 비즈 앱만 가능.** 개인 앱이면 이메일 못 켜서 KOE205 발생.

### 2-B. 키 2개 (위치 주의: [앱] → [플랫폼 키])
- **REST API 키** = Supabase **Client ID**
- REST API 키 카드의 **클라이언트 시크릿** → 코드 생성(상태 ON) →
  **"카카오 로그인" 코드** = Supabase **Client Secret** (비즈니스 인증 코드 아님)

### 2-C. 카카오 로그인 켜기 + 동의항목
- **제품 설정 → 카카오 로그인 → 일반** → 상태 **ON**
- **제품 설정 → 카카오 로그인 → 동의항목**
  - 닉네임(profile_nickname) → **필수 동의**
  - **카카오계정(이메일, account_email) → 선택 동의** ← 없으면 KOE205
    (선택 동의는 검수 없이 즉시 적용)

### 2-D. Redirect URI 등록 ⚠️ (진짜 함정)
- **[앱] → [플랫폼 키]** 에서 REST API 키의 **Redirect URI** 로 등록
  - "카카오 로그인 → 고급"의 **"로그아웃 리다이렉트 URI"가 아님!**
- 값: `https://ifjikwpmyfvzwetppnfx.supabase.co/auth/v1/callback`
- Web 플랫폼 등록을 먼저 요구하면 사이트 도메인 `https://icebrain78.github.io` 등록 후 진행

### 2-E. Supabase
- **Providers → Kakao** → Enable → Client ID(REST API 키) / Secret(카카오 로그인 코드) → **Save**

### 겪은 에러
| 에러 | 뜻 | 해결 |
|------|----|------|
| **KOE205** | 요청한 동의항목(account_email) 미설정 | 비즈 앱 전환 후 이메일 **선택 동의** |
| **KOE006** | Redirect URI 미등록 | **[앱]→[플랫폼 키]** 에 콜백 주소 등록 |

## 3. 네이버 로그인 (Edge Function 필요)

네이버는 Supabase 기본 제공이 아니라, 네이버 OAuth를 처리하고 Supabase 세션을
발급하는 **Edge Function**을 하나 배포한다. 코드는 저장소에 있음
(`supabase/functions/naver-callback/index.ts`).

### 동작 원리
```
앱 → 네이버 authorize → (code) → Edge Function(naver-callback)
  → 네이버 토큰/프로필 조회 → service_role로 유저 생성/확인
  → Supabase 매직링크 발급 → 그 링크가 앱으로 #access_token=... 실어 복귀
```

### 3-A. 네이버 개발자센터
1. [developers.naver.com](https://developers.naver.com) → 애플리케이션 등록
2. 이름 `물들임`, **사용 API: 네이버 로그인** (권한: 회원이름·이메일·별명)
3. **환경 추가 → PC 웹**
   - 서비스 URL: `https://icebrain78.github.io`
   - **Callback URL**: `https://ifjikwpmyfvzwetppnfx.supabase.co/functions/v1/naver-callback`
4. **Client ID / Client Secret** 복사

### 3-B. Edge Function 배포 (대시보드)
1. Supabase → **Edge Functions → Functions → 새 함수 생성**
2. 함수 이름: **`naver-callback`**
3. `index.ts` 기본 코드를 지우고 저장소의 함수 코드를 통째로 붙여넣기
4. **Deploy**
5. 배포 후 그 함수의 **"Verify JWT" 를 OFF** ⚠️
   (네이버가 인증 헤더 없이 브라우저로 호출하므로 켜져 있으면 막힘)
   - CLI라면: `supabase functions deploy naver-callback --no-verify-jwt`

### 3-C. 시크릿 2개 (Edge Functions → Secrets)
이름을 **정확히** (대문자+언더바):
| Name | Value |
|------|-------|
| `NAVER_CLIENT_ID` | 네이버 Client ID |
| `NAVER_CLIENT_SECRET` | 네이버 Client Secret |

> `SUPABASE_URL`, `SUPABASE_SERVICE_ROLE_KEY` 는 자동 주입되어 있음.
> 하나로 합쳐 만들지 말 것 — 반드시 **두 개의 개별 시크릿**.

### 3-D. 앱에 Client ID 넣기 (공개 OK)
`js/cloud-config.js` 의 `naverClientId` 에 네이버 **Client ID** 입력 → 커밋.
값이 채워지면 로그인 창에 **"네이버로 계속하기"** 버튼이 나타남.
(Client Secret은 절대 코드/깃허브에 넣지 말 것 — Edge Function 시크릿에만.)

### 막힐 때
- 함수 로그: Supabase → Edge Functions → naver-callback → **Logs**
- 주소창 `#error=naver_token_failed` → Client ID/Secret 또는 Callback URL 불일치
- `#error=session_failed` → service_role 문제(드묾)
- 갤러리로 안 돌아옴 → 0번 Redirect URLs 확인

## 4. 최종 — 어떤 값이 어디에 사는가

| 값 | 위치 | 공개 여부 |
|----|------|-----------|
| Supabase URL / anon key | `js/cloud-config.js` | 공개 OK |
| 네이버 Client ID | `js/cloud-config.js` (`naverClientId`) | 공개 OK |
| 구글 Client ID/Secret | Supabase Providers(Google) | 비공개(서버) |
| 카카오 REST키/시크릿 | Supabase Providers(Kakao) | 비공개(서버) |
| 네이버 Client Secret | Edge Function Secret(`NAVER_CLIENT_SECRET`) | **비공개(서버)** |
| service_role key | Supabase 자동 주입(코드에 안 넣음) | **절대 공개 금지** |

> 규칙: **Secret 류는 절대 저장소에 커밋하지 않는다.** 서버(Supabase Providers /
> Edge Function Secret)에만 저장. 공개돼도 되는 건 anon key, 각 Client **ID** 뿐.

## 5. 안드로이드 앱(Capacitor)에서의 소셜 로그인

- 웹은 리다이렉트, 앱은 **커스텀 스킴 딥링크**(`io.github.icebrain78.coloring://login-callback`)로 복귀.
- 딥링크 intent-filter는 빌드 시 워크플로가 AndroidManifest에 자동 주입.
- `@capacitor/browser` 로 시스템 브라우저를 열고, `@capacitor/app` 의 `appUrlOpen` 으로 토큰 회수.
- 앱에서 로그인하려면 0번 Redirect URLs에 딥링크가 등록돼 있어야 함(등록됨).
