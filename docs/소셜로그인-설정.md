## ③ 카카오  ✅ (완료됨) — 헷갈리는 포인트 정리

> 카카오 콘솔이 개편돼서 위치가 특이함. 아래가 **실제로 통한** 순서.

### A. 앱 만들기 + 비즈 앱 전환
1. [developers.kakao.com](https://developers.kakao.com) → 애플리케이션 추가 (이름 `물들임`)
2. **앱 아이콘 등록** (`앱 → 일반` 또는 `앱 기본 정보` → 앱 아이콘) — 비즈 전환에 필요
3. **비즈 앱 전환** — 개인 개발자는 **본인인증**만으로 무료 (사업자번호 불필요)
   - 이메일 동의를 받으려면 비즈 앱이 **반드시** 필요 (개인 앱은 이메일 못 켬 → KOE205)

### B. 키 2개 (⚠️ 위치 주의)
- **앱 → 플랫폼 키** 에서:
  - **REST API 키** = Supabase의 **Client ID**  (예: `81277b11e546198e82d2fe8b9e5b69e2`)
  - REST API 키 카드의 **클라이언트 시크릿** → 코드 생성(ON) → **카카오 로그인** 코드 = Supabase의 **Client Secret**

### C. 카카오 로그인 켜기 + 동의항목
- **제품 설정 → 카카오 로그인 → 일반** → 상태 **ON** (OpenID Connect도 ON 무방)
- **제품 설정 → 카카오 로그인 → 동의항목**:
  - 닉네임(profile_nickname) → **필수 동의**
  - **카카오계정(이메일, account_email) → 선택 동의**  ← 이거 없으면 KOE205
    (선택 동의는 검수 없이 바로 적용됨)

### D. Redirect URI 등록 (⚠️ 여기가 진짜 함정)
- **[앱] → [플랫폼 키]** 에서 REST API 키의 **Redirect URI** 로 등록
  (카카오 로그인 화면의 "로그아웃 리다이렉트 URI"가 **아님**!)
- 값: `https://ifjikwpmyfvzwetppnfx.supabase.co/auth/v1/callback`
- Web 플랫폼 등록을 먼저 요구하면: 사이트 도메인 `https://icebrain78.github.io` 등록 후 진행

### E. Supabase
- Authentication → Providers → **Kakao** → Enable
  - Client ID ← REST API 키
  - Client Secret ← 카카오 로그인 클라이언트 시크릿 코드
  - Save

### 겪은 에러 정리
| 에러 | 뜻 | 해결 |
|------|----|------|
| KOE205 | 이메일 동의항목 미설정 | 비즈 앱 전환 후 account_email 선택 동의 |
| KOE006 | Redirect URI 미등록 | [앱]→[플랫폼 키]에 콜백 주소 등록 |
